一.Redis

1.缓存穿透

查询一个不存在的数据，mysql也查询不到不会直接写入缓存，就会导致每次请求到数据库。

解决方案：

  • 缓存空值

优点：实现简单

缺点：缓存大量空值会消耗内存，可能发生不一致的问题（原来没有该数据缓存为空值，但是后续数据库中添加了，就会直接返回空值）

  • 布隆过滤器

数据结构：以bit为单位的数据，每个单元存储0或1

原理：通过多个hash函数计算散列数据，最后查询时通过相同hash函数判断是否为1，为1即存在。

优点：内存占用少，没有多余key

缺点：实现复杂，存在误判(hash碰撞问题是使用hash函数共性问题)

Redisson中有对应实现

2.缓存击穿

某个key设置了过期时间，当key过期时，恰好这时有对这个key有大量的并发请求，可能就会瞬间击穿缓存到达数据库。

解决方案：

  • 使用互斥锁（强一致性）

通过Redis的setnx命令设置互斥锁，操作成功时重建缓存，重建成功才释放锁。其他线程获取锁失败就休眠等待重试。

  • 设置逻辑过期（高可用）

通过多缓存一个过期时间字段去判断是否去设置互斥锁然后新开线程重建缓存，获取锁失败则直接返回过期数据。

3.缓存雪崩

同一时段大量的缓存key同时失效或者Redis服务宕机，导致大量请求直接到达数据库

解决方案：

  • 给不同的key的TTL添加随机值
  • 利用哨兵模式、集群模式搭建Redis集群
  • 给缓存业务添加降级限流策略（spring cloud gateway）
  • 给业务添加多级缓存

4.双写一致性

Redis作为缓存，怎么和Mysql的数据进行同步？

  • 允许延时一致的业务，采用异步通知

①使用MQ中间件，更新数据后，通知缓存删除

②采用canal中间件，不需要修改业务代码，伪装为mysql的一个从节点，canal通知读取binlog数据更新缓存。(基于mysql的binlog文件)

  • 强一致性的，采用Redisson提供的读写锁

②排他锁：独占锁(写锁)，加锁后阻塞其他线程写操作

6.数据过期策略

Redis采用惰性删除+定期删除两种策略配合使用。

  • 惰性删除

访问key的时候判断是否过期，如果过期则删除（意味着只有访问到时才会进行删除操作）

优点：减少对cpu的影响，性能好

缺点：内存占用大

  • 定期删除

每隔一段时间就对key进行检查，删除里面的过期key

①SLOW模式

这个模式是定时任务，执行频率默认为10hz(1秒执行10次)，每次不超过25ms，可以通过修改redis.conf文件hz选项来调节这个次数。

②FAST模式

执行频率不固定，但是两次间隔不低于2ms，每次耗时不超过1ms.

优点：可以有效释放占用内存

缺点：难以确定删除操作的时长和频率

3.索引概念及底层数据结构

底层数据结构（B+树）

  • 红黑树作为一种树只有两个子节点，所以数据量太大的时候会导致层级太高

  • B树因为拥有多个子节点，因此查找笑了更好，因为将层级的数据分散，因此是一颗矮胖树。

B+树不同于B树，B+数据只在叶子节点存储数据（所有非叶子节点同样会保存数据在叶子节点），非叶子节点只作为查找数据的索引，而且叶子节点是双向链表（便于区间查询）。

1.4 Spring的bean的生命周期

在Spring框架中，BeanPostProcessor接口定义了在Spring容器实例化Bean之后和在调用Bean的初始化方法之前提供回调的方法。它允许开发人员在Bean实例化和初始化过程中干预，对Bean进行自定义操作或者修改。BeanPostProcessor接口包含以下两个主要方法：

  1. postProcessBeforeInitialization(Object bean, String beanName): 在调用Bean的初始化方法之前被调用。开发人员可以在这个方法中对Bean进行修改或者增强。例如，可以在Bean初始化之前进行日志记录、权限检查、属性注入等操作。
  2. postProcessAfterInitialization(Object bean, String beanName): 在调用Bean的初始化方法之后被调用。开发人员可以在这个方法中对Bean进行进一步的定制或者处理。例如，可以在Bean初始化之后注册到其他组件、进行AOP代理等操作。

BeanPostProcessor接口的典型用途包括：

  • AOP代理：在Bean初始化之后，通过动态代理为Bean生成AOP代理，从而实现切面功能。
  • 值注入：在Bean初始化之前或之后，对Bean的属性进行注入或者修改。
  • 生命周期回调：在Bean初始化之前或之后执行特定的逻辑，如记录日志、权限检查等。

Spring框架本身提供了多个内置的BeanPostProcessor实现，如AutowiredAnnotationBeanPostProcessor、CommonAnnotationBeanPostProcessor等，用于处理自动装配、常用注解等功能。同时，开发人员也可以自定义BeanPostProcessor接口的实现类，以实现特定的定制化需求。
